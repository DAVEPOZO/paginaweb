¿Qué ES FLEXBOX?

CSS Flexbox, presentado en 2009, es un potente sistema de diseño unidimensional diseñado para ayudarte a crear páginas web adaptables y organizar elementos fácilmente. Rápidamente se convirtió en el método predilecto para diseños modernos, ya que elimina la necesidad de trucos complejos como flotantes y posicionamiento.

Flexbox le permite organizar elementos en filas o columnas, lo que simplifica el control del espaciado, la alineación y el orden dentro de un contenedor.

Uso Básico

Antes de explicar cuándo usar Flexbox y Grid, repasemos algunas propiedades que diferencian estos conceptos. Si quieres profundizar en las propiedades de Flexbox y Grid, o cómo empezar a usarlos, puedes consultar estos artículos para principiantes sobre Flexbox y Grid, respectivamente.

Supongamos que tenemos un elemento <div> en nuestro HTML que sirve como contenedor para tres elementos secundarios. Solo necesitas crear un contenedor flexible usando la propiedad. Después, cada elemento dentro de ese contenedor flexible se convierte en un elemento flexible: display: flex

El código correspondiente sería: 


<div class="contenedor">
  <div id="uno">1</div>
  <div id="dos">2</div>
  <div id="tres">3</div>
</div>

.contenedor {
  display: flex;
}

Direcciones y ejes flexibles

En nuestro contenedor flexible, podríamos definir una dirección para los elementos. Las direcciones flexibles más utilizadas son row y column. Nuestro CSS correspondiente se vería así: 


.contenedor {
  flex-direction: row | column;
}

En algunos casos, se podría establecer flex-direction en row-reverse o column-reverse. Un aspecto importante a tener en cuenta en las direcciones flexibles es el concepto de ejes.

Envolver artículos

De forma predeterminada, los elementos flexibles intentan caber en una sola línea. Puedes permitir que ocupen varias líneas usando flex-wrap: 


.contenedor {
  flex-wrap: wrap | nowrap | wrap-reverse;
}

El problema con flex-wrap, sin embargo, es que al envolverse, los elementos forman su propia línea flexible debajo de los superiores, por lo que no están perfectamente alineados con los elementos superiores. Esto se observa cuando, por ejemplo, item4 e item5 se extienden para llenar todo el espacio debajo de los elementos superiores.

Alineación y ordenamiento

Flexbox tiene muchas otras propiedades que podemos usar para crear cosas increíbles. Podemos ordenar los elementos como queramos, invertir su orden, determinar si deben crecer o contraerse, etc.

Un concepto que hace tan útil a Flexbox es la capacidad de alineación de elementos dentro del contenedor flexible. Para demostrarlo, veamos algunas propiedades de Flexbox relacionadas con la alineación, como align-self y justify-content.

Por ejemplo, imaginemos que nuestro contenedor <div> ahora tiene cuatro elementos, con el mismo ancho y alto: 


<div id="contenedor">
  <div id="uno">1</div>
  <div id="dos">2</div>
  <div id="tres">3</div>
  <div id="cuatro">4</div>
</div>

#uno {
  align-self: flex-start | flex-end | center | stretch;
}

En lugar de alinear cada elemento individualmente, supongamos que queremos alinear todos los elementos dentro del contenedor. Para ello, usaremos justify-content. Supongamos que, en lugar de cuatro, tenemos tres elementos en nuestro contenedor: 


.contenedor {
  justify-content: flex-start | flex-end | center | space-between | space-around;
}

Puedes observar lo fácil que es alinear el contenido dentro de los contenedores flexibles, ya sean elementos individuales dentro del contenedor o todos juntos. Las funciones anteriores solo muestran algunas propiedades comunes de Flexbox y lo fácil que es administrar pequeñas partes de tu diseño.

¿Qué es CSS Grid?

Mientras que Flexbox es un sistema de diseño unidimensional (que gestiona filas o columnas), CSS Grid lleva el diseño a un nuevo nivel al ser un sistema bidimensional. Esto significa que permite trabajar con filas y columnas simultáneamente, lo que lo convierte en la herramienta de diseño CSS más potente disponible actualmente.

Grid te permite crear diseños complejos y organizados sin recurrir a trucos CSS obsoletos. Es perfecto para crear diseños robustos y estructurados con un control preciso sobre ambos ejes.

Introducción a CSS Grid

Utilicemos el fragmento HTML a continuación y veamos cómo podemos obtener un diseño de cuadrícula a partir de él:

Alineación de elementos en la cuadrícula

Al igual que ahora, Grid también cuenta con propiedades que permiten alinear elementos dentro del contenedor. Un ejemplo es la propiedad justify-self. Esta propiedad alinea un elemento de la cuadrícula dentro de su celda, a lo largo del eje de la fila: 


#one {
  justify-self: start | end | center | stretch;
}

Propiedades que comparten Flexbox y Grid

Si bien Flexbox y CSS Grid difieren en muchos aspectos, comparten puntos en común importantes, empezando por su propósito principal: ambos son potentes sistemas de diseño diseñados para reemplazar técnicas antiguas y engorrosas como los flotantes y los trucos de posicionamiento. Hoy en día, simplifican e intuitivos la creación de diseños complejos y adaptables.

Tanto Flexbox como Grid admiten propiedades de alineación clave que le ayudan a controlar la posición de los elementos dentro de los contenedores:

  • justify-content — Alinea los elementos a lo largo del eje principal (horizontal en la fila de Flexbox, horizontal en la cuadrícula de forma predeterminada)
  • align-content — Administra el espaciado entre filas o columnas cuando hay espacio adicional
  • align-items — Alinea todos los elementos dentro del contenedor en el eje transversal (vertical en la fila de Flexbox)
  • align-self — Permite que los elementos individuales anulen la configuración de alineación del contenedor

Estas propiedades compartidas proporcionan consistencia y facilitan el cambio entre Flexbox y Grid según sus necesidades de diseño.

CSS Grid es para el diseño; Flexbox es para la alineación

Cuando se lanzó Flexbox por primera vez, pensamos que podría ser el mejor sistema de diseño para usar en la creación de nuestras páginas web, pero no lo fue.

Flexbox ayudó a los desarrolladores a comenzar a crear aplicaciones web más receptivas y fáciles de mantener, pero la idea principal de un sistema de diseño unidimensional no tiene sentido cuando se necesita crear un diseño más complejo.

CSS Grid nos ayudó a crear diseños de maquetación más complejos de forma bidimensional, utilizando tanto filas como columnas. Deberíamos intentar usar ambos juntos, pero con diferentes propósitos. Para el diseño, usa CSS Grid; para la alineación de los elementos, usa Flexbox. Para dominar y saber exactamente cuándo vas a necesitar y cómo usar CSS Grid, primero debes aprender los conceptos básicos y cómo funciona Flexbox, porque cuando necesites la alineación de elementos en tu aplicación, es Flexbox lo que querrás usar.

Cuándo usar CSS Flexbox

Tienes un diseño pequeño para implementar: Flexbox es ideal cuando tienes un diseño de diseño pequeño para implementar, con unas pocas filas o unas pocas columnas.

Necesitas alinear elementos: Flexbox es perfecto para eso, lo único que debemos hacer es crear un contenedor flexible usando display: flex y luego definir la dirección flexible que queremos.

Necesitas un diseño que priorice el contenido : Flexbox es el sistema de diseño ideal para crear páginas web si no sabes exactamente cómo se verá tu contenido, así que si quieres que todo encaje, Flexbox es perfecto para eso.

Claro que puedes crear toda tu aplicación usando solo Flexbox y obtener el mismo resultado que con CSS Grid. Pero para un mejor enfoque CSS, para tener una aplicación más concisa, bien escrita y fácil de mantener a largo plazo, y para crear y ajustar tu diseño a la perfección, el método ideal es usar CSS Grid.

Cuándo usar CSS Grid

Tienes un diseño complejo que implementar: En algunos casos de uso, tenemos diseños complejos que implementar, y ahí es donde la magia de CSS Grid se manifiesta. El sistema de diseño bidimensional es perfecto para crear un diseño complejo. Podemos usarlo a nuestro favor para crear páginas web más complejas y fáciles de mantener.

Necesitas un espacio entre los elementos del bloque: Otra característica muy útil en CSS Grid, que no tenemos en Flexbox, es la propiedad gap. Podemos definir el espacio entre nuestras filas o columnas fácilmente, sin tener que usar la propiedad margin, que puede causar efectos secundarios, especialmente si trabajamos con muchos puntos de interrupción.

Necesitas superponer elementos: Superponer elementos con CSS Grid es muy fácil; solo necesitas usar las propiedades grid-column y grid-row y podrás tener elementos superpuestos fácilmente. Por otro lado, con Flexbox aún necesitamos usar algunos trucos como márgenes, transformaciones o posicionamiento absoluto.

Necesita un diseño que priorice el diseño: cuando ya tiene la estructura de diseño, es más fácil construir con CSS Grid, y el sistema de diseño bidimensional nos ayuda mucho cuando podemos usar filas y columnas juntas y posicionar los elementos de la manera que queremos.
